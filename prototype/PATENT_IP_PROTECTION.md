## 2. PATENTABLE INNOVATIONS

### 2.1 Core Algorithm Patents

#### Patent Application #1: Single-Match Dating Algorithm
- **Title:** "System and Method for Sequential Single-Match Dating with Timed Decision Windows"
- **Abstract:** A dating platform algorithm that presents users with one potential match at a time, requiring a decision within a specified time window before the next match is presented.
- **Claims:**
  1. A method for presenting dating matches sequentially to a user
  2. Time-limited decision windows (24-hour periods) for match acceptance
  3. Profile visibility toggling based on match status
  4. Automatic match expiration and rotation systems

#### Patent Application #2: Mutual Match Verification System
- **Title:** "Bidirectional Match Confirmation with Synchronized Timer Reset"
- **Abstract:** A system that requires both parties to accept a match before communication is enabled, with timer synchronization upon mutual acceptance.
- **Claims:**
  1. Two-way match acceptance verification
  2. Timer reset upon mutual match confirmation
  3. Pending state management for asymmetric acceptance
  4. Automatic match cancellation for unmatched interests

#### Patent Application #3: Conversation-Based Venue Recommendation
- **Title:** "AI-Driven Date Venue Suggestions Based on Chat Content Analysis"
- **Abstract:** A system that analyzes user conversations to recommend appropriate date venues and activities.
- **Claims:**
  1. Natural language processing of chat messages for keyword extraction
  2. Interest mapping from conversation topics to venue categories
  3. Dynamic venue ranking based on shared interests
  4. Location-aware venue filtering and suggestions

### 2.2 User Experience Patents

#### Patent Application #4: Profile Visibility Management
- **Title:** "Automated Profile Visibility Control Based on Match Status"
- **Abstract:** A system that automatically hides user profiles when they have pending or active matches, preventing overlapping connections.
- **Claims:**
  1. Automatic profile hiding upon match presentation
  2. Profile restoration upon match completion/rejection
  3. User-controlled visibility overrides
  4. Match-status-based visibility states

#### Patent Application #5: Safety-First Date Planning
- **Title:** "Emergency Contact Notification System for Dating Applications"
- **Abstract:** A safety feature that automatically notifies designated emergency contacts when a user plans a date.
- **Claims:**
  1. Emergency contact registration and management
  2. Automatic notification upon date confirmation
  3. Date details sharing (location, time, match information)
  4. Check-in and safety confirmation systems

### 2.3 Business Method Patents

#### Patent Application #6: One-Match-at-a-Time Business Model
- **Title:** "Focused Dating Platform with Limited Match Presentation"
- **Abstract:** A business method for operating a dating platform that deliberately limits users to one active match at a time.
- **Claims:**
  1. User engagement model based on scarcity
  2. Quality-over-quantity matching philosophy
  3. Subscription model tied to match frequency
  4. User retention through focused interactions

## 7. IP ENFORCEMENT POLICY

### 7.1 Monitoring

OITH actively monitors for IP infringement through:
- App store monitoring for similar applications
- Web crawling for unauthorized use of marks
- Social media monitoring
- Competitor analysis

### 7.2 Enforcement Actions

Upon discovery of potential infringement:

1. **Level 1: Cease and Desist**
   - Formal letter demanding cessation of infringing activity
   - Documentation of infringement for legal record

2. **Level 2: DMCA/Platform Takedown**
   - File takedown notices with app stores
   - Report trademark violations to platforms

3. **Level 3: Legal Action**
   - File for preliminary injunction
   - Pursue damages for willful infringement
   - Seek attorney's fees recovery

## 8. EMPLOYEE IP AGREEMENTS

### 8.1 Required Agreements

All employees, contractors, and consultants must sign:

1. **Invention Assignment Agreement**
   - All work product belongs to OITH
   - Prior inventions must be disclosed
   - Cooperation in patent applications required

2. **Confidentiality Agreement**
   - Non-disclosure of proprietary information
   - Survives termination of employment
   - Covers trade secrets and business information

3. **Non-Compete Agreement** (where enforceable)
   - Restrictions on competing employment
   - Geographic and temporal limitations
   - Consideration provided for agreement

### 8.2 IP Training

All personnel receive training on:
- Identifying patentable innovations
- Protecting confidential information
- Proper use of third-party IP
- Reporting potential infringements
